# Infrastructure Setup (Web, DB, Monitoring)

## Overview
This Vagrant configuration automatically provisions three virtual machines connected via an isolated private network:
1. **Web Server** (`192.168.33.11`): runs Nginx and serves a static HTML page.
2. **Database Server** (`192.168.33.12`): runs MySQL with a sample `test_db` database using environment variables.
3. **Monitoring Server** (`192.168.33.13`): runs Prometheus to monitor the infrastructure via metrics.

---

## Setup Instructions
1. Ensure Vagrant and a compatible provider (VirtualBox) are installed on your host system.
2. Open your host terminal (PowerShell or CMD) and navigate to the project directory
3. Run the following command to boot up and provision all three nodes:
```bash
   vagrant up
   ```
## How to Test
1. Web Page Access
Open your host web browser and go to:

http://localhost:8080
Expected Result: You should see a page displaying: "Welcome to the Web Server!".

2. Database Query Validation
Connect to the web server from your host terminal via SSH:

```bash
vagrant ssh web
```
Execute the automated database test script with administrator privileges:

```bash
sudo ./db_test.sh
```
Check the persistent logs generated by the database query:

```bash
cat /var/log/db_query.log
```
Expected Result: The terminal should output the queried table records:


id   name
1    Alice
2    Bob
Disconnect from the virtual node:

```bash
exit
```
3. Monitoring Metrics Verification
Open your host web browser and navigate to the Prometheus dashboard:

http://localhost:9090
In the top menu, click on Status -> Targets.

Expected Result: You should see all three targets (prometheus, web_server_metrics, and db_server_metrics) displaying a green UP status badge.
